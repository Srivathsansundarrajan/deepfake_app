--------------------------------------------------------------------------------
a. URL / SOURCE FOR DATASET
--------------------------------------------------------------------------------

Dataset : FaceForensics++
GitHub  : https://github.com/ondyari/FaceForensics

FaceForensics++ is a forensics dataset consisting of video sequences that
have been manipulated with four automated face manipulation methods:
  - Deepfakes
  - Face2Face
  - FaceSwap
  - NeuralTextures

The dataset provides:
  - Real (original) video sequences sourced from YouTube
  - Manipulated video sequences generated by the above methods
  - Three compression levels: RAW (c0), HIGH (c23), LOW (c40)

To request access and download the dataset, follow the instructions provided
in the official FaceForensics++ GitHub repository linked above.


--------------------------------------------------------------------------------
b. SOFTWARE AND HARDWARE REQUIREMENTS
--------------------------------------------------------------------------------

SOFTWARE REQUIREMENTS
---------------------
  Operating System   : Windows 10/11, Ubuntu 18.04+, or macOS 10.15+
  Python             : 3.8 or higher (3.10 recommended)
  Package Manager    : pip (comes with Python)

  Required Python Libraries (see requirements.txt):
    - streamlit            : Web application framework for the UI
    - torch                : PyTorch deep learning framework (CPU or CUDA)
    - opencv-python-headless : Video frame extraction and image processing
    - numpy                : Numerical computations
    - gdown                : Automatic model download from Google Drive
    - insightface          : Face detection and alignment (InsightFace library)
    - onnxruntime          : ONNX model runtime (required by InsightFace)
    - timm                 : PyTorch Image Models (EfficientNet backbone)

  All dependencies can be installed via:
    pip install -r requirements.txt

HARDWARE REQUIREMENTS
---------------------
  Minimum:
    - CPU    : Any modern dual-core processor (Intel Core i3 / AMD Ryzen 3)
    - RAM    : 4 GB
    - GPU    : Not required (CPU inference is fully supported)
    - Storage: 1 GB free disk space (for models and temporary files)

  Recommended:
    - CPU    : Intel Core i5 / AMD Ryzen 5 or higher
    - RAM    : 8 GB
    - GPU    : NVIDIA GPU with CUDA support (optional, for faster inference)
    - Storage: 2 GB free disk space

  Cloud Deployment (e.g., Streamlit Community Cloud):
    - Minimum 1 GB RAM available for the application instance
    - Models are downloaded automatically from Google Drive on first run

c. DETAILED INSTRUCTIONS TO EXECUTE THE SOURCE CODE
--------------------------------------------------------------------------------

PROJECT FILE STRUCTURE
-----------------------
  deepfake_app/
  |-- app.py                  # Main Streamlit application (entry point)
  |-- detector.py             # Model loading and inference pipeline
  |-- model_architecture.py   # Phase2Model neural network definition
  |-- preprocess.py           # Video preprocessing and face extraction
  |-- requirements.txt        # Python dependency list
  |-- ReadMe.txt              # This file
  |-- models/                 # Auto-created folder for downloaded .pth files
      |-- exp1.pth            # Model 1 weights (Experiment 1 - ImageNet init)
      |-- best.pth            # Model 2 weights (Experiment 3 - Phase1 init)

STEP-BY-STEP EXECUTION INSTRUCTIONS
-------------------------------------

STEP 1 — Install Python
  Download and install Python 3.8 or higher from https://www.python.org/downloads/
  Make sure to check "Add Python to PATH" during installation (Windows).

STEP 2 — Clone or Download the Project
  If using Git:
    git clone <your-repository-url>
    cd deepfake_app

  Or download and extract the project ZIP folder, then open a terminal/command
  prompt and navigate to the project directory:
    cd path\to\deepfake_app        (Windows)
    cd path/to/deepfake_app        (Mac/Linux)

STEP 3 — (Optional) Create a Virtual Environment
  It is strongly recommended to use a virtual environment to avoid package
  conflicts with other Python projects.

    python -m venv venv

  Activate the virtual environment:
    Windows :  venv\Scripts\activate
    Mac/Linux: source venv/bin/activate

STEP 4 — Install Dependencies
  Run the following command to install all required packages:

    pip install -r requirements.txt

  NOTE: If you have an NVIDIA GPU and want GPU acceleration, install PyTorch
  with CUDA support first. Visit https://pytorch.org/get-started/locally/ to
  get the correct install command for your CUDA version, for example:
    pip install torch torchvision --index-url https://download.pytorch.org/whl/cu118

STEP 5 — Run the Application
  Launch the Streamlit web application by running:

    streamlit run app.py

  This will automatically open the application in your default web browser.
  If it does not open automatically, navigate to:
    http://localhost:8501

STEP 6 — Using the Application
  1. The application will open in your browser titled:
     "AI-Based Deepfake FaceSwap Detection"

  2. Click the "Browse files" button to upload a video file.
     Supported formats: .mp4, .avi, .mov

  3. After uploading, the video will be previewed on screen.

  4. Click the "Run Deepfake Detection" button to start analysis.

  5. The system will automatically:
     a. Download the pre-trained model weights from Google Drive (first run only)
        - exp1.pth  (Model 1 — EfficientNet-B0 with ImageNet initialization)
        - best.pth  (Model 2 — EfficientNet-B0 with Phase-1 initialization)
     b. Extract face regions from video frames using InsightFace
     c. Compute DCT (Discrete Cosine Transform) frequency heatmaps
     d. Run inference through both deep learning models

  6. Results displayed include:
     - Extracted Face Frames  : Visual preview of detected face regions
     - DCT Frequency Heatmaps : Frequency-domain artifact visualization
     - Model 1 Prediction     : REAL / FAKE label with confidence score (%)
     - Model 2 Prediction     : REAL / FAKE label with confidence score (%)

STEP 7 — Stopping the Application
  Press CTRL + C in the terminal window where Streamlit is running to stop
  the server.

TROUBLESHOOTING
---------------
  Issue   : ModuleNotFoundError for any package
  Solution: Re-run  pip install -r requirements.txt  inside the activated venv

  Issue   : Model weights not downloading
  Solution: Ensure you have a stable internet connection. The models are
            downloaded from Google Drive (~260 MB each) on the first run.
            They are saved in the models/ directory for subsequent runs.

  Issue   : "No face detected" or empty frames
  Solution: Ensure the video contains clearly visible frontal faces.
            Very low resolution or heavily occluded faces may not be detected.

  Issue   : Out of Memory (OOM) error
  Solution: Close other applications to free RAM. The app loads models
            sequentially to minimize memory usage (~300 MB per model).
            A minimum of 8 GB RAM is recommended.

  Issue   : Streamlit port already in use
  Solution: Run on a different port:  streamlit run app.py --server.port 8502
